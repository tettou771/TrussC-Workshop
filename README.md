# TrussC Workshop

TrussC 워크숍용 샘플 코드야.

테마별로 프로젝트를 나눠놨고, 각 테마의 사용법과 소스코드 후반부에 「챌린지」 코너가 있어.

먼저 실행해보고 (VSCode 등에서는 F5 키), 어떤 건지 확인한 다음 tcApp.h, tcApp.cpp를 살펴봐. 아하~ 하고 이해가 됐으면 챌린지 부분을 풀어보면 돼.

챌린지에는 정답이 없으니까 네가 생각한 대로 동작하면 OK야.

## 샘플 종류

### 00-basicGraphics

기본적인 도형 그리기야. 선을 긋거나 다각형으로 영역을 만들 수 있어.

### 01-drawShape_vs_drawStroke

Processing에서는 두꺼운 stroke에 cap을 넣어서 끝을 둥글게 하거나, 꺾인 선의 모서리를 깎을 수 있잖아. openFrameworks에서는 안 돼서 불만이었는데, ofxStroke라는 애드온을 만들어 썼었어.

TrussC에서는 그런 기능도 내장되어 있어서 그래픽 표현이 좀 더 쉬울 거야.

### 02-tween

애니메이션할 때 부드럽게 가감속하는 샘플이야.

그냥 사각형이라도 두부처럼 보이거나, UI의 기분 좋은 느낌을 움직임으로 표현할 수 있어서 꽤 중요한 요소야. 근데 tween은 외부 애드온 없이는 쓰기 어려운 경우가 많아서, TrussC에서는 내장했어.

### 03-mouse_key

마우스와 키 이벤트 처리 방법이야.

간단하니까 가볍게 훑어보면 될 것 같은데, 키 입력은 대문자로 핸들링한다는 점만 주의해. 예를 들면 if (key == 'C') 이런 식이야.

### 04-transform

좌표를 이동하거나 회전하거나 스케일링하는 샘플이야.

pushMatrix(), popMatrix() 같은 메서드는 oF나 Processing에도 있으니까 익숙할 수도 있어.

### 05-mathGenerative

sin, cos, noise를 활용한 제너러티브 그래픽스 샘플이야.

파도를 그리거나, 리사주 곡선을 그리거나, noise로 유기적인 움직임을 만들어. 크리에이티브 코딩의 묘미는 바로 여기지.

### 06-imageTexture

픽셀 단위로 이미지를 조작하는 샘플이야.

Image를 allocate하고, setColor로 픽셀을 쓰고, update로 GPU에 업로드하고, draw로 표시하는 흐름이야. 이걸 이해하면 프로시저럴 텍스처 생성이나 실시간 이미지 처리를 할 수 있게 돼.

### 07-nodeSystem

RectNode를 사용한 부모-자식 관계 샘플이야.

addChild로 부모에 자식을 추가하면 부모가 움직일 때 자식도 따라와. 로컬 좌표로 그려지니까 UI 패널이나 버튼을 만들 때 편리해. enableEvents()를 호출하면 마우스 호버나 클릭도 잡을 수 있어.

### 08-fbo

FBO(오프스크린 렌더링) 샘플이야.

begin() / end() 사이에 그린 것이 텍스처가 돼. TextureFormat 지정 방식이 oF 등과는 다른 enum이니까 주의해.

### 09-sound

beep()과 ChipSound 샘플이야.

beep()은 내장 프리셋 사운드로, 디버그나 UI 피드백에 쓸 수 있어. ChipSound는 파형, 주파수, 길이를 지정해서 직접 사운드를 만드는 구조야. 패미컴 느낌의 효과음을 간단하게 만들 수 있어.
